Researchers Expose Widespread Phishing Technique That Bypasses Modern Browser Defenses

A coalition of cybersecurity researchers from three universities published findings this week detailing a sophisticated social-engineering campaign that has successfully compromised thousands of enterprise workstations despite the presence of modern endpoint detection tools.

The technique, which the researchers have dubbed "ClickFix", relies on a deceptively simple premise: convincing users to manually execute commands they believe are harmless verification steps. Unlike traditional phishing attacks that require victims to open attachments or click suspicious links, ClickFix weaponizes the browser's native clipboard API to silently stage a payload before the victim is even aware anything has occurred.

"The genius — and the danger — is that the attacker never delivers the payload directly," said Dr. Priya Nair, lead author of the study. "The victim's own operating system executes it, triggered entirely by the victim's own keystrokes."

The attack begins when a user visits a compromised or attacker-controlled website. The page presents what appears to be a standard bot-verification widget — visually indistinguishable from the reCAPTCHA components embedded on millions of legitimate sites. When the user clicks the familiar "I'm not a robot" checkbox, the page silently invokes navigator.clipboard.writeText() with a malicious command string.

In the campaigns the researchers analyzed, that command was typically a PowerShell one-liner designed to download and execute a second-stage dropper from an attacker-controlled CDN. The commands were obfuscated using base64 encoding and executed with the -WindowStyle Hidden flag, ensuring no console window appeared to alert the victim.

What followed was the social engineering masterstroke: a modal dialog, again styled to match a familiar verification UI, instructed the user to "complete verification" by pressing a keyboard shortcut — on Windows, Win+R to open the Run dialog, followed by Ctrl+V and Enter. On macOS, the equivalent steps used Spotlight or Terminal. The entire sequence, from clipboard write to command execution, took less than fifteen seconds.

Why it works

  • The CAPTCHA widget is pixel-perfect — visually identical to Google reCAPTCHA
  • Clipboard write happens silently on checkbox click — no browser prompt on most origins
  • The "verification steps" reframe a dangerous action (run a shell command) as a routine task
  • Social pressure ("unusual activity detected") overrides the victim's scepticism
  • The payload executes inside a trusted OS process — EDR/AV telemetry often misses it
  • No malicious file is downloaded by the browser — no URL-based detection fires

Mitigations

  • User awareness: no legitimate CAPTCHA ever asks you to run shell commands
  • Browser extensions can block navigator.clipboard.writeText on suspicious origins
  • Restrict Win+R / Run dialog via Group Policy in enterprise environments
  • Monitor for clipboard-triggered process spawns (PowerShell / cmd launched with pasted args)
